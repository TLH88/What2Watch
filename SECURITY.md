# Security Policy

What2Watch was designed for deployment in a trusted local environment. It is not intended or represented as a hardened internet-facing application.

Current limitations include:

- no authentication system
- permissive CORS
- local `.env`-based secret storage
- admin functionality intended for trusted users

If you choose to deploy this project, whether locally or on the public web, you do so entirely at your own risk.

The developer provides this software on an "as is" basis and accepts no responsibility or liability for:

- security incidents
- data loss
- credential exposure
- service outages
- misconfiguration
- compatibility issues
- any direct or indirect damage resulting from use, deployment, or modification of this project

If you discover a security issue in the repository, do not open a public GitHub issue with exploit details.

Instead, report it privately to the repository owner and include:

- a short description of the problem
- affected files or endpoints
- reproduction steps
- impact assessment
